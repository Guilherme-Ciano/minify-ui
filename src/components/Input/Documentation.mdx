# Input Component Documentation

The `InputText` component allows users to input text with various configurations such as label position, icons, helper text, and error handling. It offers flexibility and customization options for text input fields within a user interface.

## Examples

These examples demonstrate different use cases and configurations of the `InputText` component, providing a versatile and user-friendly input solution within the application.

```tsx
import { InputText } from 'minify-ui';
import { FaAddressCard, FaCalendar } from 'react-icons/fa';

// Default input
<InputText label="Username" placeholder="Enter your username" />

// Input with icons
<InputText leftIcon={<FaAddressCard />} placeholder="Enter your address" />
<InputText rightIcon={<FaCalendar />} type="date" />

// Input with helper text
<InputText label="Password" placeholder="Enter your password" helper="Password must be at least 8 characters long." />

// Input with error handling
<InputText label="Email" placeholder="Enter your email" errors="Invalid email address" />
```

## Props

The `InputText` component accepts the following props:

- `size`: Specifies the size of the input field. Options include `'sm'`, `'md'`, and `'lg'`. Defaults to `'md'`.
- `label`: Specifies the label text for the input field.
- `placeholder`: Specifies the placeholder text for the input field.
- `labelPosition`: Specifies the position of the label relative to the input field. Options include `'outside'` and `'inside'`. Defaults to `'outside'`.
- `helper`: Provides helper text to assist users with input field usage.
- `leftIcon`: Specifies an icon to be displayed on the left side of the input field.
- `rightIcon`: Specifies an icon to be displayed on the right side of the input field.
- `disabled`: Disables the input field if set to `true`.
- `errors`: Specifies error messages to be displayed below the input field.
- `success`: Indicates a successful input field state.
- `value`: Specifies the value of the input field.
- `onChange`: Handles input field value changes.
- `type`: Specifies the type of input. Options include `'number'`, `'email'`, `'password'`, `'search'`, `'text'`, and `'date'`. Defaults to `'text'`.
