# Button Component Documentation

The `Button` component provides a customizable button element for user interaction. It supports various visual variants such as default, primary, secondary, danger, base, and text buttons, along with additional features like size, hover effect, shadow, and disabled state.

## Props

The `Button` component accepts the following props:

- `type`: Specifies the visual variant of the button. Default is `'Default'`.
- `size`: Specifies the size of the button. Default is `'sm'`.
- `hover`: Specifies whether the button should have a hover effect. Default is `false`.
- `shadow`: Specifies whether the button should have a shadow effect. Default is `false`.
- `disabled`: Specifies whether the button is disabled. Default is `false`.
- `onClick`: Function to handle button click events.
- `children`: JSX content to be displayed inside the button.
- `id`: Specifies the id attribute of the button.
- `icon`: Icon component to be displayed inside the button.
- `trailIcon`: Icon component to be displayed after the button content.
- `className`: Additional CSS class(es) to apply to the button.
